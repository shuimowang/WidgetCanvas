# WebDAV 多设备同步

WidgetCanvas 可以通过一个用户指定的 WebDAV 目录同步组件。该功能默认关闭，不需要注册 WidgetCanvas 账号，也不会经过 WidgetCanvas 项目服务器。

## 配置

1. 在 WebDAV 服务中创建一个专用目录，例如 `WidgetCanvas`。
2. 从系统托盘打开“管理中心”。
3. 填写该目录的完整 WebDAV 地址、用户名和密码；推荐使用 HTTPS 和服务商提供的应用专用密码。
4. 先点击“测试连接”，成功后点击“立即同步”。
5. 如果需要持续同步，开启右上角的“自动同步”并保存。

地址必须指向已经存在的目录，并以 HTTP 或 HTTPS 开头。WidgetCanvas 会在该目录内维护一个文件：

```text
widgetcanvas-sync-v1.json
```

服务器需要支持标准 WebDAV `PROPFIND`，以及对同步文件的 `GET` 和 `PUT`。当前账号密码验证使用 HTTP Basic Authentication，因此应优先使用 HTTPS。

## 同步内容

会同步：

- 每个组件的稳定 ID；
- 完整单文件 HTML；
- 通过 `host.state` 保存的组件数据，例如便签、待办和组件设置。

不会同步：

- 命名画布列表、当前画布，以及组件的画布归属、坐标、尺寸和层叠顺序；
- 独立窗口的位置、尺寸、置顶和贴边隐藏设置；
- WebView2 缓存、Cookie 和登录状态；
- WidgetCanvas 应用设置、快捷键、日志和本地文件；
- WebDAV 地址、用户名和密码。

从另一台设备首次收到的组件会进入组件库，不会直接出现在画布上，避免不同显示器尺寸导致组件重叠。已有组件在本机的位置和窗口选项始终保留。

## 自动同步时机

开启自动同步后，应用会在以下时机检查：

- 启动后；
- 组件数据保存并静置约 15 秒后；
- 应用运行期间每 5 分钟。

网络失败不会删除或回滚本地数据，错误会写入日志。下次自动检查或手动点击“立即同步”时会继续尝试。

## 删除与冲突

WidgetCanvas 使用上一次成功同步的本地基线进行三方合并：

- 只在一台设备发生的新增、修改或删除会正常传播；
- 一台设备删除、另一台设备同时修改时，保留修改后的组件；
- 两台设备同时修改同一个组件且内容不同时，保留本机版本，并把远端版本作为带有“来自其他设备”标题的独立组件放入组件库；
- 标题冲突会自动追加序号，确保 `--widget "标题"` 仍能准确匹配。

上传使用 WebDAV 返回的 ETag 做条件写入。如果上传前远端刚被另一台设备更新，应用会重新读取并合并，而不是直接覆盖。

## 本地安全

WebDAV 密码使用 Windows DPAPI 按当前用户加密后保存在：

```text
%LocalAppData%\浮岛\Settings\settings.json
```

同步基线保存在：

```text
%LocalAppData%\浮岛\Sync\webdav-base.json
```

DPAPI 密文不能直接复制到另一台电脑使用；每台设备都需要自行填写凭据。同步文件包含组件 HTML 和组件状态，如果组件中保存了敏感内容，应确认所使用 WebDAV 服务的可信度。
